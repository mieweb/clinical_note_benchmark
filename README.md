# Clinical Note Benchmark

This repository provides tools and scripts for benchmarking clinical note generation and evaluation using various models and prompts. It is designed to facilitate the generation, evaluation, and analysis of clinical notes, supporting reproducible research in medical NLP.

## Links
- **[project charter](https://docs.google.com/document/d/17kTRlaJ-WLMO4AmKrWsXW68A1P4XvN4vpRuMnmAoxrc/edit?tab=t.dwxviclbjl4p)
- **[preliminary research](https://docs.google.com/document/d/1GwqnsjJfnyMzu1XKPRD2jVr80PWbRBQv33b-MkxHt68/edit?tab=t.0#heading=h.il1fimp7oz65)

## Features
- **Note Generation**: Generate synthetic clinical notes using different models and prompts.
- **Evaluation**: Evaluate generated notes against reference standards using ROUGE metrics and AI-based evaluation.
- **Data Management**: Organize results, standards, and metadata for easy analysis.
- **Extensible**: Easily add new models, prompts, or evaluation methods.

## Directory Structure

## Getting Started

### Prerequisites
- Python 3.8+
- [pip](https://pip.pypa.io/en/stable/)
- (Optional) Virtual environment (recommended)

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/amandamarg/clinical_note_benchmark.git
   cd clinical_note_benchmark
   ```
2. (Optional) Create and activate a virtual environment:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

### Usage
- **Generate Notes**:
  ```bash
  python generate.py
  ```
- **Set Standards**:
  ```bash
  python set_standards.py
  ```
- **Evaluate Notes**:
  ```bash
  python evaluate.py
  ```
- **Generate Plots**:
  ```bash
  python plot.py
  ```
  
- **Custom Runs**:
  Modify `IDXS`, model names, prompt names, or other args in the scripts as needed.

## Data
- **augmented-clinical-notes/augmented_notes_30K.jsonl**: Source conversations for note generation.
- **standards/**: Reference notes for evaluation.
- **results/**: Generated notes and evaluation outputs. Each subdirectory in `results/` is named for the corresponding idx in `augmented_notes_30K.jsonl` and contains text/json. The notes generated for a transcript at a particular idx using some model and prompt will be located in the file `gen_note.txt` under `results/{idx}/{model}/{prompt}/{timestamp}`. Json files with evaluation information, such as rouge scores (`rouge-1.json`, `rouge-2.json`, and `rouge-l.json`) and AI-driven evaluations (`ai-eval.json`) for a given note are contained in the same directory as the generated note they correspond to.
- **plots/**: Plots generated by `plot.py` script

## Results
- The results included in the directory are for a small sample of the dataset corresponding with the following idxs: `224`, `431`, `562`, `619`, `958`, `1380`, `1716`, `1834`, `2021`, `3026`, `3058`, `3093`, `3293`, `3931`, `4129`, `155216`. For each idx, three notes were generated each of the `g1` prompt and the `g2` prompt in order to compare the similarity of generated notes accross multiple runs. We found that for the most part, there was no significant variation accross different runs, however there were a few cases where this was not true and we did find differences in the concepts that were added or missing. For notes generated using `g1`, we found variations in what information was considered "missing" (i.e. included in the reference note but not the generated note) across multiple runs for `1716` and what information was "added" (i.e. included in the generated note but not the reference note) across multiple runs for `3093`. For notes generated using `g2`, we found variations in missing what was missing across multiple runs for `155216`, `1716`, `2021`, `224`, `3026` and `3091`, and what was added for `958`.

## Extending
